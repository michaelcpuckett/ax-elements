Navigational (or sectioning) landmarks comprise the document outline. The document outline allows users of assistive technologies to skip to or skip over sections of the document. Landmarks also assist robots in understanding which parts of the document are important, and how information is grouped. The section role represents a discrete structural unit of the document that contributes to the document outline, whose content is all related in some way. Section roles can be nested. The region role represents some part of the document that does not comprise the document's structure but is still significant enough to contribute to the document outline. Here, sectioning roles and heading levels are implied via calculation, and can be overridden. The outermost landmark automatically gets the main role (of which there can only be one). Descendant landmarks of the main role automatically get the section role. Descendant landmarks of section roles automatically get the region role. The main role represents the unique structural unit of the document that contributes to the document outline, and whose content consitutes the current document's main content. This means it should not include repeated site-oriented content. A banner landmark identifies site-oriented content at the beginning of each page within a website. Site-oriented content typically includes things such as the logo or identity of the site sponsor, and site-specific search tool. The banner landmark is a unique top-level role that identifies a section of the document that does not consitute the main content. It should appear before the main sectioning element. The contentinfo role represents the unique footer structural unit of the document that contributes to the document outline, but whose content is repeated on multiple documents and as such does not consitute the current document's main content. It should appear after the main sectioning element. The navigation role represents a structural unit of the document that contributes to the document outline, whose content is links for navigating the document or related documents. This lankmark only accepts links. Non-link elements are removed.
